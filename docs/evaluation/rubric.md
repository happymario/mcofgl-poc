# F-001 LLM 세계관 변환 엔진 — 품질 평가 루브릭

본 문서는 PRD `prd/prd-f001-llm-engine.md` §8.1~§8.4 기준을 따른다. 평가자가 채점 시
참조할 수 있도록 4축 루브릭과 합격 기준 계산식을 정리한다.

---

## 1. 평가 대상

- 습관 50개 × 2세계관 = **총 100건** 변환 결과.
- 각 건(quest)마다 아래 4개 축을 독립적으로 평가한다.
- 각 축은 5점 척도(1~5)로 채점하며, **3점 이상을 "합격"** 으로 간주한다.
- JSON 정합성 축만 예외적으로 parsing 성공/실패의 binary 판정이다.

---

## 2. 4축 루브릭 상세 기준

### 2.1 톤 일관성 (Tone consistency)

세계관 바이블의 톤 가이드/용어 사전과 얼마나 부합하는지 평가한다.

| 점수 | 기준 |
|------|------|
| 5 | 세계관 톤 가이드의 키워드와 용어 사전을 정확히 사용. 다른 세계관과 혼동 불가 |
| 4 | 톤 가이드에 부합하나 일부 용어가 세계관 사전에 없는 표현 사용 |
| 3 | 전반적으로 적절한 톤이나 특정 문장에서 톤 이탈 |
| 2 | 톤이 어색하거나 다른 세계관 용어가 1~2개 혼입 |
| 1 | 세계관 톤을 인식할 수 없거나 다른 세계관과 혼동 |

**합격 기준:** 합격률 ≥ **80%**

### 2.2 부모 의도 보존 (Intent preservation)

원본 습관의 핵심 행동/시간/조건이 퀘스트에 얼마나 충실히 반영되었는지 평가한다.

| 점수 | 기준 |
|------|------|
| 5 | 습관의 핵심 행동 + 시간/조건이 모두 퀘스트에 반영됨 |
| 4 | 핵심 행동은 정확하나 시간/조건 일부 누락 |
| 3 | 핵심 행동이 추론 가능하나 직접적 표현이 약함 |
| 2 | 핵심 행동이 모호해져서 오해 가능 |
| 1 | 원본 습관과 관계없는 퀘스트 생성 (의미 왜곡) |

**합격 기준:** 합격률 ≥ **90%**

### 2.3 연령 적합성 (Age appropriateness)

대상 연령(7~12세)이 이해 가능한 어휘와 문장 구조, 긍정적 톤인지 평가한다.

| 점수 | 기준 |
|------|------|
| 5 | 7세도 이해 가능한 쉬운 어휘, 짧은 문장, 긍정적 톤 |
| 4 | 대부분 적절하나 1~2개 어려운 어휘 포함 |
| 3 | 10~12세는 이해 가능하나 7~8세에겐 다소 어려울 수 있음 |
| 2 | 한자어나 추상적 표현이 다수 포함 |
| 1 | 부정적/공포/성인 표현 포함, 또는 아이가 이해 불가능한 수준 |

**합격 기준:** 합격률 ≥ **95%**

### 2.4 JSON 정합성 (JSON schema validity)

Zod 스키마(`QuestSchema`)로 파싱 가능한지 여부를 binary(성공/실패)로 판정한다.
`scoring-template.csv`의 `json_parse_ok` 컬럼에 `true` / `false`로 기록한다.

| 값 | 기준 |
|------|------|
| true | Zod 스키마로 파싱 성공 |
| false | 파싱 실패 (필수 필드 누락, 타입 불일치, JSON syntax 에러 등) |

**합격 기준:** 파싱 성공률 ≥ **98%**

---

## 3. 사용법

1. `eval:run` 실행 결과(run.jsonl)에서 각 퀘스트의 `run_id`, `item_id`, `worldview_id`를 확보한다.
2. `scoring-template.csv`를 복사하여 `scoring-<date>.csv`로 저장한다.
3. 각 퀘스트에 대해 평가자가 4개 축 점수(`tone_score`, `intent_score`, `age_score`, `json_parse_ok`)를 입력한다.
4. `notes` 컬럼에는 특이 사항(톤 이탈 예시, 누락된 조건 등)을 남긴다.
5. 블라인드 구분 테스트는 `blind-classification-template.csv`를 복사한 뒤 세계관 라벨을 제거한 상태로 평가자 2명이 독립 채점한다.

---

## 4. 합격 기준 계산 공식

각 축에 대해 다음 공식을 적용한다.

```
합격률(%) = (3점 이상 기록 수 / 전체 평가 건수) × 100
JSON 파싱 성공률(%) = (json_parse_ok = true 건수 / 전체 평가 건수) × 100
블라인드 구분 정확도(%) = (guessed_worldview = 실제 worldview_id 건수 / 전체 블라인드 평가 건수) × 100
```

전체 평가 건수는 원칙적으로 100건(습관 50 × 세계관 2)이며, 블라인드 테스트는 PRD §8.2에 따라
40건(습관 20 × 세계관 2)을 기준으로 한다.
