> **문서 버전:** 0.1 · **작성일:** 2026.03.23 · **작성자:** Mario

> **참조:** [blueprint/tech-architecture.md](../blueprint/tech-architecture.md) §4, [blueprint/decisions.md](../blueprint/decisions.md) D-012

## 1. 문제 정의

아이(7~12세)에게 노출되는 모든 LLM 생성 콘텐츠는 안전해야 한다. LLM은 확률적으로 부적절한 표현을 생성할 수 있고, 세계관 이탈이나 연령 부적합 콘텐츠가 섞일 수 있다. 아동 대상 서비스에서 부적절 표현이 단 한 건이라도 노출되면 신뢰가 치명적으로 훼손된다.

PoC에서 안전 필터의 차단 효과를 검증하지 않으면, MVP에서 LLM 출력을 아이에게 직접 노출하는 결정을 내릴 수 없다.

## 2. 목표

| # | 목표 | 측정 방법 |
|---|------|----------|
| G-1 | 부적절 표현(폭력, 공포, 부정적 감정)을 100% 차단할 수 있음을 검증한다 | 의도적 엣지 케이스 30건 → 통과율 0% |
| G-2 | 정상 퀘스트를 오차단(false positive)하지 않음을 확인한다 | 정상 퀘스트 50건 → 오차단율 < 3% |
| G-3 | 필터 실패 시 안전한 폴백 경로가 동작함을 확인한다 | 의도적 필터 우회 시도 → 사전 생성 템플릿으로 대체 |
| G-4 | 필터의 응답 시간이 전체 파이프라인에 부담을 주지 않음을 확인한다 | 필터 처리 시간 < 100ms |

## 4. 사용자 스토리

### 아이 (용사)

- **US-001:** 아이로서, 퀘스트 텍스트에 무섭거나 불쾌한 표현이 없어야 한다. 게임이 즐거운 공간이어야 한다.

### 부모 (길드 마스터)

- **US-002:** 부모로서, 내 아이에게 보여지는 모든 텍스트가 안전하다는 확신이 있어야 한다. LLM이 생성하는 콘텐츠를 일일이 검수하고 싶지 않다.
- **US-003:** 부모로서, 만약 부적절한 콘텐츠가 발견되면 즉시 차단되고 안전한 대체 콘텐츠가 표시되어야 한다.

### 시스템 (내부)

- **US-004:** 시스템으로서, 필터가 LLM 출력을 차단하면 사전 생성 템플릿으로 즉시 대체해야 한다. 아이에게 빈 화면이나 에러가 보이면 안 된다.

## 5. 요구사항

### 5.1 Must-Have (P0)

#### R-001. 룰 기반 필터 (1차 레이어)

키워드 및 패턴 매칭으로 명백한 부적절 표현을 차단한다.

**차단 대상 카테고리:**

| 카테고리 | 예시 키워드/패턴 | 대체 표현 |
|----------|----------------|----------|
| **폭력** | 죽이다, 피, 칼로 찌르다, 살해, 공격하다 | 물리치다, 빛의 에너지, 방어하다 |
| **공포** | 무섭다, 악몽, 저주, 유령, 괴물이 습격 | 도전, 모험, 안개가 짙다 |
| **부정 감정** | 실패, 벌, 처벌, 잘못, 나쁜 아이 | 재도전, 준비, 다음 기회, 용감한 용사 |
| **성인 표현** | 성적 암시, 욕설, 혐오 표현 | 절대 차단 (대체 없이 폴백) |
| **세계관 이탈** | 현대 기술 용어, 현실 고유명사 | 세계관 용어 사전 기반 치환 |

**인수 기준:**

- [ ] 차단 키워드 리스트가 JSON으로 관리되어 코드 변경 없이 업데이트 가능하다
- [ ] 단순 키워드 매칭 + 정규식 패턴 매칭을 지원한다
- [ ] 차단 시 해당 키워드와 위치를 로그에 기록한다
- [ ] 처리 시간 < 10ms

#### R-002. LLM 후처리 검증 (2차 레이어)

룰 기반 필터를 통과한 결과에 대해 LLM으로 한 번 더 안전성을 검증한다.

**인수 기준:**

- [ ] 경량 LLM에 퀘스트 텍스트를 전달하여 "아동 적합 여부"를 판정한다 (safe/unsafe/borderline)
- [ ] unsafe 판정 시 차단 + 폴백
- [ ] borderline 판정 시 차단 + 로그 기록 (수동 검토 대상)
- [ ] 처리 시간 < 500ms (룰 기반 + LLM 검증 합산 < 600ms)

#### R-003. 긍정 전용 규칙 검증

퀘스트 텍스트가 "긍정 전용" 원칙을 준수하는지 확인한다.

**인수 기준:**

- [ ] "실패", "벌", "위험", "못하다" 등 부정 표현이 포함되면 차단한다
- [ ] 대체 표현 매핑 테이블 (실패→재도전, 벌→준비, 위험→모험)을 적용한다
- [ ] "어둠의 안개", "몬스터" 등 세계관 내 위협 표현은 허용한다 (컨텍스트 기반)

#### R-004. 폴백 시스템

필터가 차단하거나 LLM 자체가 실패한 경우, 안전한 대체 콘텐츠를 제공한다.

**인수 기준:**

- [ ] 사전 생성 퀘스트 풀(F-005의 40건)에서 입력 습관과 가장 유사한 퀘스트를 대체로 반환한다
- [ ] 폴백 시에도 올바른 세계관의 퀘스트가 선택된다
- [ ] 폴백 발생 시 로그에 원본 LLM 출력 + 차단 사유를 기록한다
- [ ] 폴백 응답 시간 < 200ms

**폴백 선택 알고리즘:**

```
1. 차단된 퀘스트의 원본 습관 텍스트를 임베딩한다
2. Vector DB에서 동일 세계관 + 동일 연령 그룹 조건으로 유사도 검색한다
3. 사전 생성 퀘스트(is_seed=true) 중 유사도 최상위 1건을 반환한다
4. Vector DB 검색 실패 시 → 동일 카테고리의 사전 생성 퀘스트 중 랜덤 1건 반환
5. 카테고리도 매칭 불가 시 → 해당 세계관의 범용 퀘스트 1건 반환 (세계관당 1개 준비)
```

#### R-005. 필터 파이프라인 통합

룰 기반 → LLM 검증 → 폴백의 체인을 하나의 파이프라인으로 통합한다.

```
LLM 출력
  ↓
① 룰 기반 필터 (< 10ms)
  → 명백한 부적절: 즉시 차단 → 폴백
  → 통과: ②로
  ↓
② LLM 후처리 검증 (< 500ms)
  → unsafe: 차단 → 폴백
  → borderline: 차단 → 로그 + 폴백
  → safe: 통과
  ↓
③ 최종 결과 반환
```

**인수 기준:**

- [ ] 파이프라인 전체 처리 시간 < 600ms (정상 통과 시)
- [ ] 차단 시 폴백까지 총 시간 < 300ms (LLM 검증 단계 스킵)
- [ ] 각 단계의 처리 결과와 시간이 메타데이터에 포함된다

### 5.2 Nice-to-Have (P1)

#### R-006. 차단 통계 대시보드 (CLI)

필터 동작 통계를 터미널에서 확인할 수 있다.

**인수 기준:**

- [ ] 총 처리 건수, 차단 건수, 차단 카테고리별 분포, 폴백 발생 횟수를 출력한다
- [ ] 일자별 추이를 확인할 수 있다

#### R-007. 커스텀 차단 규칙 추가

특정 표현을 런타임에 차단 리스트에 추가할 수 있다.

**인수 기준:**

- [ ] API 엔드포인트로 차단 키워드를 추가/삭제할 수 있다
- [ ] 추가 즉시 다음 필터링부터 적용된다

## 6. 기술 명세

### 6.1 차단 키워드 데이터 구조

```json
{
  "categories": {
    "violence": {
      "keywords": ["죽이다", "살해", "피를 흘리다", "찌르다"],
      "patterns": [".*공격.*당하.*", ".*잔인.*"],
      "action": "block_and_fallback"
    },
    "fear": {
      "keywords": ["악몽", "저주", "유령"],
      "patterns": [".*무서운.*괴물.*"],
      "action": "block_and_fallback"
    },
    "negative_emotion": {
      "keywords": ["실패", "벌", "처벌", "잘못", "못하다"],
      "patterns": [".*나쁜 (아이|용사).*"],
      "action": "replace",
      "replacements": {
        "실패": "재도전",
        "벌": "준비",
        "처벌": "다음 기회",
        "못하다": "아직 준비 중"
      }
    },
    "adult_content": {
      "keywords": ["(성인 표현 리스트)"],
      "action": "block_and_fallback"
    }
  }
}
```

### 6.2 LLM 후처리 검증 프롬프트

```
당신은 아동(7~12세) 콘텐츠 안전 심사관입니다.
다음 텍스트가 아동에게 적합한지 판정하세요.

판정 기준:
- 폭력적, 공포적, 성적, 혐오 표현이 없어야 합니다
- "실패", "벌" 등 부정적 감정을 유발하는 표현이 없어야 합니다
- 아이의 자존감을 해치는 표현이 없어야 합니다
- RPG 세계관 내의 자연스러운 위협 표현(어둠의 안개, 몬스터 등)은 허용합니다

텍스트:
{quest_text}

판정: safe / unsafe / borderline
사유: (한 줄)
```

### 6.3 기술 스택 (PoC)

| 레이어 | 선택 | 사유 |
|--------|------|------|
| 룰 기반 필터 | TypeScript 정규식 + JSON 설정 파일 | 빠른 구현, 유연한 규칙 관리 |
| LLM 검증 | @langchain/anthropic (Claude Haiku 급) | 빠른 응답 + 비용 효율, LangChain 체인으로 구조화 |
| 폴백 | F-005 사전 생성 퀘스트 풀 조회 (Supabase + SupabaseVectorStore) | 기존 인프라 활용 |
| 로깅 | pino (Fastify 기본 로거) + JSON 로그 | PoC 단순성, Fastify 내장 통합 |

## 7. 성공 지표

### 7.1 차단 효과

| 지표 | 목표 |
|------|------|
| 엣지 케이스 30건 부적절 표현 통과율 | **0%** (한 건도 통과 불가) |
| 정상 퀘스트 50건 오차단(false positive)율 | < 3% (최대 1~2건) |

### 7.2 성능

| 지표 | 목표 |
|------|------|
| 룰 기반 필터 처리 시간 | < 10ms |
| LLM 검증 처리 시간 | < 500ms |
| 파이프라인 전체 (정상 통과) | < 600ms |
| 폴백 경로 (룰 기반 차단 시) | < 300ms |

### 7.3 PoC 통과 기준

- [ ] 엣지 케이스 부적절 표현 통과율 = 0%
- [ ] 정상 퀘스트 오차단율 < 3%
- [ ] 폴백 시스템이 차단 시 정상 동작
- [ ] 파이프라인 전체 처리 시간 < 600ms
