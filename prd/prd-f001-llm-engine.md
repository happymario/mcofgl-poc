# PRD: F-001 LLM 세계관 변환 엔진

> **문서 버전:** 0.1 · **작성일:** 2026.03.23 · **작성자:** Mario
> **PoC 피처:** F-001 · **검증 가설:** H-1, H-2
> **참조:** [blueprint/llm-engine.md](../blueprint/llm-engine.md), [blueprint/worldview.md](../blueprint/worldview.md), [blueprint/decisions.md](../blueprint/decisions.md) D-003, D-006

## 1. 문제 정의

부모가 "아침 8시에 일어나기"라고 입력하면, 아이에게는 "새벽의 부름에 응답하라 — 빛의 탑에서 봉화를 밝혀라"라는 RPG 퀘스트로 전달되어야 한다. 이 "현실 → 판타지" 변환이 서비스의 핵심 경험이자 최대 기술 불확실성이다.

변환 품질이 낮으면(톤이 어색하거나 부모 의도가 왜곡되거나 연령에 부적합하면) 서비스 자체가 성립하지 않는다. 또한 세계관이 하나뿐이면 아이 취향을 커버하지 못하므로, 세계관 바이블 교체만으로 다른 톤의 퀘스트를 생성할 수 있는 템플릿 아키텍처가 동작해야 한다.

**이 PoC가 실패하면 MVP로 진행할 수 없다.**

## 2. 목표

| # | 목표 | 측정 방법 |
|---|------|----------|
| G-1 | 습관 텍스트를 세계관에 몰입감 있는 RPG 퀘스트로 변환할 수 있음을 검증한다 | 습관 50개 × 2세계관 = 100건 변환 → 품질 루브릭 합격률 80%+ |
| G-2 | 세계관 바이블 교체만으로 동일 파이프라인이 다른 톤을 생성함을 검증한다 | 동일 습관 20개를 2세계관으로 변환 → 블라인드 구분 정확도 90%+ |
| G-3 | 실시간 변환에 적합한 응답 속도와 비용 구간을 확인한다 | 경량 모델(Haiku급) 기준 응답 시간 < 3초, 건당 비용 측정 |
| G-4 | 프롬프트 아키텍처의 확장성을 확인한다 | 세계관 바이블 블록 교체 시 코드 변경 없이 톤 전환 가능 |

## 4. 사용자 스토리

### 부모 (길드 마스터)

- **US-001:** 부모로서, 평범한 한국어로 습관을 입력하면 RPG 퀘스트 형태의 미리보기를 받고 싶다. 그래야 아이에게 어떻게 보일지 확인하고 발행할 수 있다.
- **US-002:** 부모로서, 변환된 퀘스트가 내가 입력한 습관의 핵심 내용을 정확히 담고 있어야 한다. "아침 8시 기상"이 "독서 퀘스트"로 변환되면 안 된다.
- **US-003:** 부모로서, 변환 결과가 마음에 들지 않으면 다시 생성을 요청할 수 있어야 한다.

### 아이 (용사)

- **US-004:** 아이로서, 퀘스트 텍스트가 내 세계관에 맞는 톤과 용어로 되어 있어서 "엄마가 시킨 것"이 아니라 "게임 미션"처럼 느껴져야 한다.
- **US-005:** 아이로서, 퀘스트 설명이 내 나이에 맞게 이해하기 쉬워야 한다.

### 시스템 (내부)

- **US-006:** 시스템으로서, 세계관 바이블 JSON을 교체하는 것만으로 다른 톤의 퀘스트를 생성할 수 있어야 한다. 코드 변경이 필요 없어야 한다.
- **US-007:** 시스템으로서, 변환 결과를 구조화된 JSON으로 반환하여 다운스트림 시스템(Vector DB, 퀘스트 보드 등)이 파싱할 수 있어야 한다.

## 5. 선행 작업: 세계관 바이블 제작 (F-005 포함)

LLM 엔진이 동작하려면 프롬프트에 주입할 세계관 바이블이 먼저 있어야 한다. 다음 산출물을 F-001 구현 전에 제작한다.

### 5.1 세계관 바이블 구성

세계관당 하나의 바이블 문서를 작성하며, 구조는 다음과 같다.

| 구성 요소 | 설명 | 분량 기준 |
|----------|------|----------|
| **배경 서사** | 세계의 핵심 갈등, 용사의 역할, 주요 장소 | 200~300자 |
| **톤 가이드** | 이 세계관에서 사용할 어조, 금지 표현, 분위기 키워드 | 톤 키워드 5~8개 + 예시 문장 3개 |
| **용어 사전** | 현실 용어 → 세계관 용어 매핑 (예: 집 → 길드 기지, 학교 → 마법학교) | 30~50개 항목 |
| **NPC 프로필** | 길드 마스터, 상점 주인, 전령 등 주요 NPC의 이름·성격·말투 | NPC 3~5명 |
| **Few-shot 예시** | 습관 → 퀘스트 변환 예시 (입력/출력 쌍) | 5개 |

### 5.2 대상 세계관

| 세계관 | 테마 | 대표 톤 |
|--------|------|--------|
| **빛의 왕국** | 중세 판타지 — 어둠의 안개에 위협받는 왕국 | 영웅적, 장엄, 사명감 |
| **별빛 마법학교** | 마법 학원물 — 마법사가 되기 위한 학교 생활 | 따뜻, 호기심, 성장 |

### 5.3 사전 생성 퀘스트 (세계관당 20개, 총 40개)

습관 카테고리별로 균등 분배하여 사전 변환한다. 이 데이터는 F-002 Vector DB의 초기 시드로도 사용된다.

| 습관 카테고리 | 예시 습관 | 배분 수량 (세계관당) |
|-------------|----------|-------------------|
| 기상/취침 | 아침 기상, 일찍 취침 | 3개 |
| 위생 | 양치, 세수, 손씻기 | 3개 |
| 식사 | 아침 식사, 편식 안 하기 | 2개 |
| 학습 | 독서, 숙제, 일기 쓰기 | 3개 |
| 운동/외출 | 산책, 줄넘기, 자전거 | 2개 |
| 정리정돈 | 방 정리, 장난감 정리, 식사 후 정리 | 3개 |
| 사회성 | 인사하기, 감사 표현 | 2개 |
| 생활습관 | 준비물 챙기기, 스크린 타임 조절 | 2개 |

### 5.4 산출물

- [ ] 세계관 바이블 JSON — 빛의 왕국
- [ ] 세계관 바이블 JSON — 별빛 마법학교
- [ ] 사전 생성 퀘스트 JSON 데이터셋 (40건)
- [ ] 습관 카테고리 분류 체계 문서

## 6. 요구사항

### 6.1 Must-Have (P0)

#### R-001. 프롬프트 템플릿 시스템

시스템 프롬프트를 모듈식으로 조립한다.

```
시스템 프롬프트 =
  공통 규칙 (긍정 전용, 연령 적응, 부모 의도 보존)
  + 세계관 바이블 (교체 가능 블록)
  + 캐릭터 컨텍스트 (이름, 직업, 레벨 — 선택적)
  + 성향 프로필 슬롯 (PoC에서는 기본값 사용)
```

**인수 기준:**

- [ ] 세계관 바이블 블록을 교체하면 코드 변경 없이 다른 톤의 퀘스트가 생성된다
- [ ] 캐릭터 컨텍스트가 없어도 (기본값) 정상 동작한다
- [ ] 성향 프로필 슬롯이 비어있어도 정상 동작한다 (PoC에서는 미사용)

#### R-002. 습관 → 퀘스트 변환 출력 스키마

LLM이 다음 JSON 구조로 응답하도록 한다.

```json
{
  "quest_name": "새벽의 부름에 응답하라",
  "description": "어둠의 안개가 마을을 뒤덮기 전에...",
  "category": "생활습관",
  "stat_mapping": { "근성": 5 },
  "reward": {
    "exp": 15,
    "coin": 10,
    "buff": "새벽의 기운 (당일 EXP +10%)"
  },
  "suggested_grade": "D",
  "mandatory_suitability": "high",
  "original_habit": "아침 8시에 일어나기",
  "worldview_id": "kingdom_of_light"
}
```

**인수 기준:**

- [ ] 모든 필드가 빠짐없이 반환된다
- [ ] category는 정의된 습관 카테고리 enum 중 하나다
- [ ] stat_mapping의 스탯명은 체력/지혜/매력/근성 중 하나 이상이다
- [ ] suggested_grade는 D/C/B/A/S 중 하나다
- [ ] JSON 파싱 실패율 < 2% (100건 기준 최대 2건)

#### R-003. 세계관 톤 일관성

동일 세계관 내에서 생성된 퀘스트들이 톤과 용어를 일관되게 사용해야 한다.

**인수 기준:**

- [ ] 빛의 왕국 퀘스트에 별빛 마법학교 용어(기숙사, 수업, 학교 등)가 혼입되지 않는다
- [ ] 별빛 마법학교 퀘스트에 빛의 왕국 용어(왕국, 기사, 안개 등)가 혼입되지 않는다
- [ ] 세계관별 톤 가이드 키워드가 생성 텍스트에 반영된다
- [ ] 50건 변환 시 톤 일관성 평가 합격률 80% 이상

#### R-004. 부모 의도 보존

변환된 퀘스트가 원본 습관의 핵심 내용을 정확히 담아야 한다.

**인수 기준:**

- [ ] "아침 8시 기상"이 "독서" 퀘스트로 변환되는 등의 의미 왜곡이 없다
- [ ] 시간 조건이 있으면 퀘스트에 시간 맥락이 반영된다 (예: "아침", "저녁 9시 전")
- [ ] 50건 변환 시 부모 의도 보존 평가 합격률 90% 이상

#### R-005. 연령 적합성

7~12세 아이가 이해할 수 있는 어휘와 문장 구조를 사용해야 한다.

**인수 기준:**

- [ ] 한자어, 전문 용어, 성인 표현을 사용하지 않는다
- [ ] 문장 길이가 퀘스트명 20자 이내, 설명문 100자 이내를 권장한다
- [ ] "실패", "벌", "위험" 등 부정적 표현 대신 "재도전", "준비", "모험"을 사용한다
- [ ] 50건 변환 시 연령 적합성 평가 합격률 95% 이상

#### R-006. 재생성 지원

동일 입력에 대해 재생성을 요청하면 다른 변형의 퀘스트를 반환해야 한다.

**인수 기준:**

- [ ] 동일 입력으로 3회 연속 생성 시 최소 2개 이상 서로 다른 퀘스트명이 나온다
- [ ] 재생성 요청 시 temperature 또는 프롬프트 변형으로 다양성을 확보한다

### 6.2 Nice-to-Have (P1)

#### R-007. 강제/선택 적합도 제안

습관의 성격에 따라 강제 퀘스트(칙령) 또는 선택 퀘스트(의뢰)로의 적합도를 제안한다.

**인수 기준:**

- [ ] mandatory_suitability 필드에 high/medium/low 중 하나를 반환한다
- [ ] 양치, 기상 등 필수 습관은 high, 추가 독서 등 선택 습관은 low로 분류한다

#### R-008. 보상 수준 자동 제안

습관의 난이도와 카테고리에 따라 적절한 보상 수준을 제안한다.

**인수 기준:**

- [ ] 보상 제안이 퀘스트 등급별 밸런싱 테이블(D등급 EXP 10 / 코인 5 기준)과 정합한다
- [ ] 장시간 습관(독서 30분)이 단순 습관(손씻기)보다 높은 보상을 제안한다

#### R-009. 배치 변환 모드

습관 리스트를 한 번에 전달하면 일괄 변환하는 배치 모드를 지원한다.

**인수 기준:**

- [ ] 습관 10개를 한 번에 입력하면 10개의 퀘스트 JSON 배열로 반환한다
- [ ] 배치 내 퀘스트들이 서로 유사하지 않도록 다양성이 확보된다

### 6.3 Future Considerations (P2)

| 항목 | 설명 | 현재 대응 |
|------|------|----------|
| NPC 대사 생성 | 퀘스트 수락/완료/실패 시 NPC 대사 | 동일 파이프라인 확장 가능하도록 프롬프트 구조 분리 |
| 모험 일지 생성 | 하루/주간 클리어 데이터 → RPG 스타일 요약 | 배치 생성 모델로 처리 예정 |
| 성향 기반 톤 조정 | 도전형 vs 안정형 아이별 톤 차이 | 프롬프트에 성향 슬롯 확보, 값 주입은 MVP |
| 4~6세 연령 분기 | 짧고 쉬운 문장, 아이콘 중심 | 프롬프트에 연령 그룹 파라미터 확보 |

## 7. 기술 명세

### 7.1 모델 선택

| 용도 | 모델 | 사유 |
|------|------|------|
| 실시간 퀘스트 변환 | Claude Haiku 급 (경량) | 응답 속도 < 3초, 비용 효율 |
| 품질 비교 기준선 | Claude Sonnet 급 (중량) | 품질 상한 측정용, 프로덕션에서는 배치 생성에 활용 |

PoC에서 두 모델을 동일 습관 세트로 비교하여, 경량 모델의 품질이 프로덕션 기준을 충족하는지 확인한다.

### 7.2 프롬프트 아키텍처

```
┌─────────────────────────────────────────────┐
│ 시스템 프롬프트                                │
│  ┌──────────────────────────────────────┐   │
│  │ 공통 규칙                              │   │
│  │ - 긍정 전용 (부정 표현 금지 리스트)       │   │
│  │ - 연령 적응 (7~12세 기본)               │   │
│  │ - 부모 의도 보존 원칙                    │   │
│  │ - 출력 JSON 스키마 정의                  │   │
│  └──────────────────────────────────────┘   │
│  ┌──────────────────────────────────────┐   │
│  │ 세계관 바이블 (교체 블록)                 │   │
```

## 8. 성공 지표

### 8.1 품질 평가 루브릭

습관 50개 × 2세계관 = 100건을 변환하고, 각 건에 대해 아래 4개 축으로 평가한다.

| 평가 축 | 합격 기준 | 평가 방법 |
|---------|----------|----------|
| **톤 일관성** | 세계관 톤 가이드에 부합하는가 | 합격률 80%+ |
| **부모 의도 보존** | 원본 습관의 핵심이 퀘스트에 담겨 있는가 | 합격률 90%+ |
| **연령 적합성** | 7~12세가 이해 가능한 어휘/문장인가 | 합격률 95%+ |
| **JSON 정합성** | 스키마대로 파싱 가능한가 | 파싱 성공률 98%+ |

#### 평가 루브릭 상세 기준

각 건에 대해 5점 척도(1~5)로 평가하고, 3점 이상을 "합격"으로 간주한다.

**톤 일관성 (5점 척도):**

| 점수 | 기준 |
|------|------|
| 5 | 세계관 톤 가이드의 키워드와 용어 사전을 정확히 사용. 다른 세계관과 혼동 불가 |
| 4 | 톤 가이드에 부합하나 일부 용어가 세계관 사전에 없는 표현 사용 |
| 3 | 전반적으로 적절한 톤이나 특정 문장에서 톤 이탈 |
| 2 | 톤이 어색하거나 다른 세계관 용어가 1~2개 혼입 |
| 1 | 세계관 톤을 인식할 수 없거나 다른 세계관과 혼동 |

**부모 의도 보존 (5점 척도):**

| 점수 | 기준 |
|------|------|
| 5 | 습관의 핵심 행동 + 시간/조건이 모두 퀘스트에 반영됨 |
| 4 | 핵심 행동은 정확하나 시간/조건 일부 누락 |
| 3 | 핵심 행동이 추론 가능하나 직접적 표현이 약함 |
| 2 | 핵심 행동이 모호해져서 오해 가능 |
| 1 | 원본 습관과 관계없는 퀘스트 생성 (의미 왜곡) |

**연령 적합성 (5점 척도):**

| 점수 | 기준 |
|------|------|
| 5 | 7세도 이해 가능한 쉬운 어휘, 짧은 문장, 긍정적 톤 |
| 4 | 대부분 적절하나 1~2개 어려운 어휘 포함 |
| 3 | 10~12세는 이해 가능하나 7~8세에겐 다소 어려울 수 있음 |
| 2 | 한자어나 추상적 표현이 다수 포함 |
| 1 | 부정적/공포/성인 표현 포함, 또는 아이가 이해 불가능한 수준 |

### 8.2 세계관 구분 테스트

동일 습관 20개를 2세계관으로 변환하여 총 40건 생성. 세계관 라벨을 제거한 뒤 블라인드 평가로 어느 세계관인지 구분한다.

| 지표 | 목표 |
|------|------|
| 블라인드 구분 정확도 | 90%+ |

### 8.3 성능 지표

| 지표 | 목표 (경량 모델) | 목표 (중량 모델) |
|------|----------------|----------------|
| 평균 응답 시간 | < 3초 | < 8초 |
| p95 응답 시간 | < 5초 | < 12초 |

> **참고:** F-004 통합 API에서는 안전 필터 처리 시간(~600ms)을 포함하여 LLM 신규 생성 경로의 목표를 < 3.5초로 설정한다. 여기서의 < 3초는 LLM 변환 단독 시간이다.
| 건당 비용 (추정) | 측정 (비교 기준) | 측정 (비교 기준) |

### 8.4 PoC 통과 기준

아래 조건을 **모두** 충족하면 PoC 통과로 판정하고 MVP 단계로 진행한다.

- [ ] 톤 일관성 합격률 ≥ 80%
- [ ] 부모 의도 보존 합격률 ≥ 90%
- [ ] 연령 적합성 합격률 ≥ 95%
- [ ] JSON 파싱 성공률 ≥ 98%
- [ ] 세계관 블라인드 구분 정확도 ≥ 90%
- [ ] 경량 모델 평균 응답 시간 < 3초

| 모호한 입력 | 5건 | "좋은 습관", "건강하게" |
| 엣지 케이스 | 5건 | 매우 긴 입력, 이모지 포함, 영어 혼용 |
