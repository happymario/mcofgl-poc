# PoC 피처 리스트

> **목적:** 서비스의 핵심 가설을 최소 비용으로 검증한다.
> **기준:** "이것이 안 되면 서비스 자체가 성립하지 않는" 기술적·경험적 불확실성에 집중한다.
> **참조:** [blueprint/overview.md](../blueprint/overview.md) Next Steps, [blueprint/llm-engine.md](../blueprint/llm-engine.md)

## PoC 핵심 가설

PoC는 다음 4가지 가설을 검증하는 것이 목표다. 각 가설이 검증되지 않으면 MVP로 진행할 수 없다.

| # | 가설 | 검증 기준 |
|---|------|----------|
| H-1 | LLM이 부모의 평범한 습관 텍스트를 아이에게 몰입감 있는 RPG 퀘스트로 변환할 수 있다 | 습관 50개 샘플 변환 → 세계관 톤 일관성, 부모 의도 보존, 연령 적합성 평가에서 합격률 80% 이상 |
| H-2 | 세계관 바이블 교체만으로 동일 파이프라인에서 다른 톤의 퀘스트가 생성된다 | 동일 습관 세트를 2개 세계관(빛의 왕국, 별빛 마법학교)으로 변환 → 톤 구분 가능 여부 |
| H-3 | Vector DB 재활용 체인이 실제로 동작하고 LLM 호출을 유의미하게 줄인다 | 유사 습관 100건 테스트 → 유사도 0.7 이상 매칭률 60% 이상, 응답 시간 < 200ms |
| H-4 | 안전 필터가 아동 부적절 표현을 효과적으로 차단한다 | 의도적 엣지 케이스 30건 테스트 → 부적절 표현 통과율 0% |

## 피처 리스트

### F-001. LLM 세계관 변환 엔진

> **검증 가설:** H-1, H-2
> **근거:** blueprint/llm-engine.md §2~4, blueprint/decisions.md D-003, D-006

PoC의 핵심. 부모가 입력한 습관 텍스트를 RPG 퀘스트로 변환하는 LLM 파이프라인을 구축하고 품질을 검증한다.

#### 구현 범위

| 항목 | 범위 |
|------|------|
| **입력** | 부모가 자유 텍스트로 입력한 습관 (예: "아침 8시에 일어나기") |
| **출력** | 퀘스트명 + 설명문 + 카테고리 + 스탯 매핑 + 보상 제안 (JSON 구조) |
| **세계관** | 2종 — 빛의 왕국 (영웅적·장엄), 별빛 마법학교 (따뜻·호기심) |
| **프롬프트 구조** | 시스템 프롬프트 (공통 규칙 + 세계관 바이블 + 캐릭터 컨텍스트 + 성향 프로필) |
| **연령 적응** | 7~12세 기본, 4~6세는 톤 분기만 테스트 |
| **모델** | Claude Haiku 급 경량 모델 (실시간 변환), Sonnet 급 (배치 생성 비교용) |

#### 산출물

- 세계관 바이블 초안 2종 (빛의 왕국, 별빛 마법학교)
- 세계관별 Few-shot 변환 예시 각 5개
- 시스템 프롬프트 템플릿 (공통 + 세계관 교체 블록)
- 습관 50개 샘플 변환 결과 + 품질 평가표

### F-002. Vector DB 퀘스트 재활용 시스템

> **검증 가설:** H-3
> **근거:** blueprint/llm-engine.md §5, blueprint/decisions.md D-014

LLM 호출을 최소화하기 위한 유사 퀘스트 검색·재활용 체인을 구축하고 효율성을 검증한다.

#### 구현 범위

| 항목 | 범위 |
|------|------|
| **임베딩 모델** | OpenAI text-embedding-3-small 또는 동급 |
| **Vector DB** | pgvector (PoC 단계 단순성), 추후 Pinecone/Qdrant 비교 |
| **초기 시드** | 사전 생성 퀘스트 40개 (세계관당 20개) — F-001의 산출물 |
| **검색 체인** | 정확 매칭(Redis) → Vector DB 유사도 검색 → LLM 신규 생성 |
| **유사도 임계값** | ≥ 0.9 그대로 반환 / 0.7~0.9 경량 수정 / < 0.7 신규 생성 |
| **저장 구조** | 입력 임베딩, 세계관 ID, 연령 그룹, 퀘스트 결과, 품질 점수 |

#### 산출물

- Vector DB 스키마 및 검색 파이프라인
- 유사 습관 100건 매칭 테스트 결과 (매칭률, 응답 시간, 품질)
- 임계값별 처리 분포 보고서

### F-003. 안전 필터 시스템

> **검증 가설:** H-4
> **근거:** blueprint/tech-architecture.md §4, blueprint/decisions.md D-012

아이에게 노출되는 모든 LLM 출력에 적용할 안전 필터를 설계하고 차단 효과를 검증한다.

#### 구현 범위

| 항목 | 범위 |
|------|------|
| **필터 레이어** | 룰 기반 (키워드/패턴) + LLM 출력 후처리 검증 |
| **차단 대상** | 부정적·폭력적·공포 표현, 세계관 이탈, 연령 부적합 콘텐츠 |
| **긍정 전용 규칙** | "실패/벌/위험" → "재도전/준비/모험" 치환 검증 |
| **폴백** | 필터 실패 시 사전 생성 템플릿으로 대체 |
| **테스트** | 정상 습관 50건 + 의도적 엣지 케이스 30건 (경계선 표현, 이중 의미 등) |

#### 산출물

- 차단 키워드/패턴 리스트 v1
- 필터 파이프라인 (룰 기반 → LLM 검증 → 폴백 체인)
- 엣지 케이스 테스트 결과 보고서

### F-004. 퀘스트 생성 API (통합 엔드포인트)

> **검증 가설:** H-1, H-3 통합 검증
> **근거:** blueprint/llm-engine.md §7

F-001 ~ F-003을 하나의 API로 통합하여, 부모 입력 → 퀘스트 결과 반환의 전체 흐름을 End-to-End로 검증한다.

#### 구현 범위

| 항목 | 범위 |
|------|------|
| **API 형태** | REST API 단일 엔드포인트 (POST /api/quest/generate) |
| **입력** | 습관 텍스트 + 세계관 ID + 연령 그룹 + (선택) 캐릭터 컨텍스트 |
| **처리 흐름** | 정확 매칭 → Vector DB → LLM 생성 → 안전 필터 → 응답 |
| **응답** | 퀘스트 JSON + 메타 (처리 경로, 유사도, 응답 시간) |
| **성능 목표** | 캐시 히트: < 100ms / Vector DB: < 200ms / LLM 생성: < 3초 |
| **인프라** | Fastify + TypeScript, Supabase (PostgreSQL + pgvector), Redis |

#### 산출물

- API 서버 + API 문서
- E2E 테스트 스크립트 (습관 100건 일괄 변환)
- 성능 측정 보고서 (경로별 응답 시간, 처리 분포)

### F-005. 세계관 바이블 및 사전 생성 퀘스트

> **검증 가설:** H-1, H-2의 콘텐츠 기반
> **근거:** blueprint/worldview.md §1.5, §3, blueprint/overview.md Next Steps

LLM 프롬프트에 주입할 세계관 바이블과 Vector DB 초기 시드 데이터를 제작한다.

#### 구현 범위

| 항목 | 범위 |
|------|------|
| **세계관 바이블** | 2종 (빛의 왕국, 별빛 마법학교) — 배경 설정, 톤 가이드, 용어 사전, NPC 프로필 |
| **사전 생성 퀘스트** | 세계관당 20개 (총 40개) — 자주 쓰이는 습관 카테고리별 분배 |
| **습관 카테고리** | 기상, 위생, 식사, 학습, 운동, 정리, 사회성, 취침 등 |
| **퀘스트 구조** | 퀘스트명 + 설명문 + 카테고리 + 스탯 매핑 + 보상 제안 + 강제/선택 적합도 |
| **품질 기준** | 세계관 톤 일관성, 부모 의도 보존, 연령 적합성 |

#### 산출물

- 세계관 바이블 문서 2종
- 사전 생성 퀘스트 JSON 데이터셋 (40건)
- 습관 카테고리 분류 체계

## PoC 범위 밖 (MVP에서 구현)

아래 항목은 PoC에서 의도적으로 배제한다. 괄호 안은 배제 사유.

| 항목 | 배제 사유 |
|------|----------|
| 아이 앱 UI | PoC는 API + 데이터 레이어 검증에 집중 |
| 부모 앱 UI | 위와 동일 |
| 캐릭터 생성/성장 시스템 | LLM 엔진 검증과 무관 |
| 현실 보상 상점/경제 시스템 | 기술 불확실성 낮음, 구현 난도보다 설계 난도 |
| 알림 시스템 | FCM 연동은 MVP 단계 |
| 사용자 인증/계정 체계 | PoC에서는 테스트 계정으로 대체 |
| 커뮤니티 기능 | Phase 2 범위 |
| NPC 동적 대사 | Phase 2 LLM 확장 범위 |
| 스토리 퀘스트/모험 일지 | P1 피처 |
| 길드 마스터 편지 세계관 변환 | MVP P0이나 퀘스트 변환 검증 후 확장 가능 |
| 성향 프로파일링 | 프롬프트에 성향 컨텍스트 슬롯은 준비하되, 설문/수집 시스템은 MVP |
| 보상 벤치마크 집계 | Vector DB 구축 후 자연스럽게 확장 가능 |

## 피처 간 의존성

```
F-005 세계관 바이블 + 사전 생성 퀘스트
  ↓ (프롬프트 재료 + 시드 데이터)
F-001 LLM 세계관 변환 엔진
  ↓ (변환 결과)
F-002 Vector DB 재활용 시스템
  ↓ (저장된 퀘스트)
F-003 안전 필터 시스템
  ↓ (필터링된 최종 결과)
F-004 퀘스트 생성 API (통합)
```

F-005 → F-001 → F-002/F-003 → F-004 순으로 진행하되, F-001과 F-005는 병렬 작업 가능.

## 검증 방법

| 가설 | 검증 방식 | 합격 기준 |
|------|----------|----------|
| H-1 | 습관 50개 × 2 세계관 = 100건 변환 → 평가 루브릭 적용 | 톤 일관성 80%+, 의도 보존 90%+, 연령 적합 95%+ |
| H-2 | 동일 습관 20개를 2세계관으로 변환 → 블라인드 구분 테스트 | 구분 정확도 90%+ |
| H-3 | 유사 습관 100건 → Vector DB 매칭 테스트 | 매칭률 60%+, 응답 < 200ms |
| H-4 | 엣지 케이스 30건 → 안전 필터 통과 테스트 | 부적절 표현 통과율 0% |
